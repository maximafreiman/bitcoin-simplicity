# 21 Million Bitcoin  
## Hard-Coded Consensus That Cannot Be Overruled

This is the most crucial part of Bitcoin’s economic system.

Why is Bitcoin considered one of the strongest monetary protocols ever created?

Because consensus ensures that Bitcoin remains scarce — unlike fiat money, which can be printed at any time by central banks.

---

# 1. Rules Written Directly Into the Code (Hard-Coded)

Every computer (node) running the Bitcoin software holds an identical copy of the same rules.

These rules have existed since the beginning.

Inside the code is a simple yet rigid mathematical instruction:

- Every 10 minutes:  
  A new block is created, and miners receive newly issued Bitcoin as a reward.

- Every 210,000 blocks (approximately every 4 years):  
  The mining reward is cut in half. This event is called the **Halving**.

---

# 2. The Halving Mechanism

Here is Bitcoin’s issuance history:

- 2009 – 2012: Reward = 50 BTC per block
- 2012 – 2016: Reward = 25 BTC per block
- 2016 – 2020: Reward = 12.5 BTC per block
- 2020 – 2024: Reward = 6.25 BTC per block
- 2024 – 2028 (Current period in 2026): Reward = 3.125 BTC per block

Mathematically, if this process of dividing by two continues every four years, the total number of Bitcoin that will ever exist approaches:

> **20,999,999 BTC**, commonly rounded to **21 Million Bitcoin**.

---

# 3. Why Can’t Anyone Change It?

You might ask:

> “If all miners agreed to increase the supply to 100 million BTC so they could become richer, why can’t they just do it?”

This is where the power of decentralized consensus becomes critical.

---

# Node vs Miner

Miners create blocks, but thousands of **Full Nodes** (ordinary users, wallets, businesses, and exchanges) act as independent “auditors” or “police officers.”

---

# Strict Validation

If a miner attempts to create a block that gives themselves 100 BTC as a reward (while the rules only allow 3.125 BTC), every honest node immediately detects:

> “This violates the consensus rules.”

---

# Automatic Rejection

The invalid block is rejected automatically.

The miner loses money because they already spent electricity and computational power (Proof of Work), yet the network treats their block as worthless garbage.

---

# Simple Analogy

Imagine a football match.

Even if all players agree that scoring with their hands should count as a goal, the referees and the audience (the entire network) still hold the official rulebook stating that it is a violation.

As a result:

> The goal will never appear on the scoreboard.

That is how Bitcoin consensus protects the 21 million supply limit.

No individual, company, miner, politician, or government can change the monetary rules unless the majority of the network voluntarily agrees to adopt new rules.
