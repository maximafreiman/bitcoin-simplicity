# The Basic Protocol Curriculum
> Bitcoin is Money. A Digital Peer-to-Peer System with Hard-Coded Consensus.

> **Goal:** Understanding the immutable laws of the 21 million supply through simplification.

- **Writer:** Maxima Freiman  
- **Text Auditor:** Bitcoin Satoshicat  

| No | Theme | Main Topics | Militant Insight (The "Why") |
|:---|:---|:---|:---|
| 1 | **Bitcoin Fundamentals** | Whitepaper, History of Cypherpunks, Basics of Austrian Economics. | Understanding that Bitcoin is a solution to the failure of the fiat monetary system (Central Banking). |
| 2 | **Consensus** | Byzantine Generals Problem, Proof of Work (PoW), Fork Rules, 21 Million Cap. | Consensus is not subjective democracy; consensus is mathematical validation that cannot be bribed. |
| 3 | **Transactions** | UTXO Model, Script (Locking/Unlocking), Fees, Mempool, PSBT, RBF. | UTXOs are sovereignty. Every sat spent must have a clear and valid lineage. |
| 4 | **Blocks** | Block Header, Hash, TXID, Merkle Root, Reorgs. | Dissecting the block structure as the heartbeat of an economy that cannot be manipulated. |
| 5 | **Cryptography (Current Focus)** | CSPRNG, ECC (secp256k1), SHA256, RIPEMD-160, Base58 (Legacy), Bech32 (SegWit), Bech32m (Taproot). | Mathematics is the only law that recognizes neither borders nor censorship. |
| 6 | **Mining** | Difficulty Adjustment, Poisson Distribution, Supply and Halving, Block Subsidy, Mining Pools, Stratum V2, 51% Attack Vectors. | Ensuring the physical security of the network remains decentralized and resistant to state coercion. |
| 7 | **Node & P2P** | Distributed Systems, Full Node vs Pruned Node, IBD (Initial Block Download), UASF (BIP148), Peer Discovery. | "Don't Trust, Verify." Running your own node is the only path to sovereignty. |
| 8 | **The Upper Layers (Upscaling)** | Lightning Network (Layer 2), Fedi (Layer 3). | Scaling Bitcoin without sacrificing the integrity of Layer 1 as the settlement layer. |
| 9 | **Bitcoin Software** | Bitcoin Core vs Knots, GPG Verification, Reproducible Builds, Gitian. | Ensuring the code being run is truly the code agreed upon by the community. |
| 10 | **Decentralized Governance Protocol** | BIP (Bitcoin Improvement Proposals). | Understanding how rules evolve without destroying existing consensus. |
