# The Sovereign Protocoler Curriculum
> **Status:** Bitcoin is Money. Everything else is a database.  
> **Goal:** Understanding the immutable laws of the 21 Million.

- **Writer:** Maxima Freiman  
- **Text Auditor:** Bitcoin Satoshicat  

| No | Theme | Core Topics | Militant Insight (The "Why") |
|:---|:---|:---|:---|
| 1 | **Bitcoin Fundamentals** | Whitepaper, Cypherpunk History, Austrian Economics basics | To understand that Bitcoin is a solution to the failure of the fiat monetary system (Central Banking). |
| 2 | **Consensus** | Byzantine Generals Problem, Proof of Work (PoW), Fork Rules, 21 Million Cap | Consensus is not democracy; consensus is mathematical validation that cannot be bribed. |
| 3 | **Transactions** | UTXO Model, Script (Locking/Unlocking), Fee Market, Mempool | UTXO is sovereignty; every satoshi must have a clear and valid lineage. |
| 4 | **Blocks (Current Focus)** | Block Header, Coinbase Transaction, Merkle Tree, Block Subsidy vs Fees, Reorgs | Dissecting block structure as the heartbeat of an economy that cannot be manipulated. |
| 5 | **Cryptography** | SHA-256, ECC (secp256k1), ECDSA vs Schnorr, Taproot (BIP341) | Mathematics is the only law that recognizes no borders or censorship. |
| 6 | **Mining** | Difficulty Adjustment, Poisson Distribution, Stratum V2, 51% Attack Vectors | Ensuring the physical security of the network remains decentralized and resistant to state coercion. |
| 7 | **Node & P2P** | Full Node vs Pruned, IBD (Initial Block Download), UASF (BIP148), Peer Discovery | "Don't Trust, Verify." Running your own node is the only path to independence. |
| 8 | **Layer 2 (Scaling)** | Lightning Network (HTLCs), Ark Protocol, RGB, Client-side Validation | Scaling without compromising the sanctity of Layer 1 integrity. |
| 9 | **Privacy & Fungibility** | CoinJoin, Tapscript, Output Billability, Preventing Chain Analysis | Privacy is not about hiding crime, but about protecting individual freedom. |
| 10 | **Bitcoin Software** | Bitcoin Core vs Knots, GPG Verification, Reproducible Builds, Gitian | Ensuring the code you run is truly the code agreed upon by the community. |
| 11 | **Protocol Governance** | BIP Process (BIP 8/9), Soft Fork vs Hard Fork, Social Consensus | Understanding how rules evolve without destroying existing consensus. |
